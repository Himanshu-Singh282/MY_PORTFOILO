SQL PROJECT ON PIZZA SALE

INTRODUCTION

This SQL-based project dives deep into the analytics of a fictional pizza delivery business. Using structured query language, I explored data stored across four relational tables — orders, order_details, pizzas, and pizza_types — to uncover meaningful business insights.
The analysis was performed in three phases:
1.Basic Analysis included counting orders, calculating total revenue, and identifying top-selling pizzas and sizes.
2.Intermediate Queries involved joining multiple tables to analyze pizza categories, hourly order distribution, and day-wise averages.
3.Advanced Insights focused on revenue contributions, cumulative sales trends, and top-performing pizza types by category.
Through this project, I demonstrated strong command over SQL concepts such as joins, aggregation, subqueries, window functions, and date/time manipulation. The goal was to simulate a real-world data analysis workflow, turning raw transactional data into useful insights that can guide decision-making in a business environment. This project not only showcases my technical skills but also reflects my ability to think analytically and communicate results clearly through well-structured queries and interpretation.
